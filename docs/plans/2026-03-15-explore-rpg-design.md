# Explore RPG — 포트폴리오 탐험 게임 디자인

## 개요

포트폴리오 사이트에 2D 탑다운 RPG 스타일의 인터랙티브 탐험 페이지를 추가한다. 방문자가 픽셀아트 창고를 돌아다니며 프로젝트, 스킬, 히스토리, 블로그 글을 게임처럼 탐험할 수 있다.

- **경로**: `/[locale]/play`
- **유형**: 기존 사이트에 추가되는 별도 페이지 (기존 페이지 변경 없음)
- **진입점**: 홈페이지 CTA 버튼 + 네비게이션 메뉴 항목

## 기술 스택

| 영역 | 기술 | 비고 |
|------|------|------|
| 렌더링 | `@pixi/react` (최신) | React 선언형 PixiJS 통합 |
| 카메라 | `pixi-viewport` | 캐릭터 추적, 월드 바운드 클램핑 |
| AI (선택적) | `@huggingface/transformers` | WebGPU, Qwen3-0.6B-ONNX, q4f16 |
| TTS (선택적) | `kokoro-js` | 86MB, 영어 24 음성 |
| 맵 에디터 | Tiled Map Editor | JSON 타일맵 export |
| 에셋 | 무료 픽셀아트 타일셋 | kenney.nl 또는 itch.io |
| UI 오버레이 | React DOM | 대화창, 상세 패널, 미니맵 |

### context7 검증 완료 항목

- `@pixi/react`: `<Application>`, `useTick()`, 포인터 이벤트, Context 게임 상태 관리
- `pixi-viewport`: `viewport.follow(target)`, `viewport.clamp()`, 월드 크기 설정
- `@huggingface/transformers`: `pipeline("text-generation", model, { device: "webgpu", dtype: "q4f16" })`, `TextStreamer`, Web Worker 패턴

## 비주얼 스타일

- **레트로 픽셀아트** (16x16 또는 32x32 타일)
- 기존 포트폴리오의 모던한 톤과 대비되는 게임 세계 → 강한 인상
- 다크/라이트 모드 연동:
  - 다크모드: 밤 분위기, 랜턴 조명 효과 강조
  - 라이트모드: 창문에서 햇빛 들어오는 낮 분위기

## 맵 구조

로비를 중심으로 4방향에 테마별 방이 연결된다.

```
                    ┌─────────────┐
                    │   스킬 방    │
                    │  (도구 선반) │
                    └──────┬──────┘
                           │
┌─────────────┐    ┌───────┴──────┐    ┌─────────────┐
│  히스토리 방  │────│    로비      │────│ 프로젝트 방  │
│  (액자/게시판)│    │  (입구/NPC)  │    │ (작업대/PC)  │
└─────────────┘    └───────┬──────┘    └─────────────┘
                           │
                    ┌──────┴──────┐
                    │    서재     │
                    │ (블로그/책) │
                    └─────────────┘
```

### 각 방 상세

#### 로비 (시작점, 중앙)
- 안내 NPC: 조작법 안내 + 각 방 소개
- 벽에 프로필 사진/자기소개 액자
- 4방향 문 (라벨 표시)
- AI NPC (선택적): "AI 모드 활성화" opt-in

#### 프로젝트 방 (오른쪽)
- 데이터 소스: `content/projects.json`
- 프로젝트당 작업대 1개 + 모니터 오브젝트
- 상호작용: 프로젝트명, 설명, 기술스택 태그, 웹사이트/GitHub 링크

#### 스킬 방 (위쪽)
- 데이터 소스: `content/about.json` → skills
- 카테고리별 선반 (Frontend / Backend / Tools)
- 각 아이템에 스킬명 표시

#### 히스토리 방 (왼쪽)
- 데이터 소스: `content/about.json` → timeline
- 벽면에 시간순 액자 배치
- 상호작용: 날짜, 제목, 상세 설명

#### 서재 (아래쪽)
- 데이터 소스: MDX 블로그 포스트 (`content/posts/{locale}/`)
- 책장에 포스트가 책으로 꽂혀 있음
- 상호작용: 제목, 발췌, "블로그에서 읽기" 링크 → 실제 블로그 페이지 이동

### 확장성
- 데이터(JSON, MDX)에서 오브젝트를 동적 생성 → 콘텐츠 추가 시 맵 자동 확장
- 오브젝트 수 초과 시 방 크기 자동 확장 또는 페이지네이션(책장 넘기기 등)

## 캐릭터

- 16x16 또는 32x32 픽셀 스프라이트
- 4방향 걷기 애니메이션 (상/하/좌/우 각 3~4프레임)
- 대기 시 idle 애니메이션 (숨쉬기 느낌)

## 환경 연출

- **조명**: 캐릭터 주변 원형 라이팅, 오브젝트 근처 따뜻한 빛
- **파티클**: 먼지 파티클 천천히 떠다님 (PixiJS ParticleContainer)
- **타일 애니메이션**: 모니터 깜빡임, 촛불 흔들림
- **방 전환**: 문 진입 시 페이드아웃 → 페이드인

## 상호작용 시스템

### 접근 감지
- 오브젝트 근처 접근 시 `[Space]` 아이콘 + 바운스 애니메이션
- 상호작용 가능 오브젝트에 미세한 glow 이펙트

### 정보 표시 (하이브리드 방식)
1. **1차**: RPG 대화창 (하단) — 짧은 소개 타자기 효과로 출력
2. **2차**: "자세히 보기" 선택 시 → 게임 UI 톤의 상세 패널

### 대화창 조작
- Space/Enter: 다음 대화
- 클릭: 타자기 효과 스킵 → 전체 텍스트 즉시 표시

## 모바일 대응

- 화면 하단 가상 D-pad (방향키) + A 버튼 (상호작용)
- 터치로 타일 탭 시 해당 위치로 자동 이동
- 반응형 Canvas 스케일링

## 사운드 (선택적)

- BGM: lo-fi 또는 8bit 배경음악 (기본 음소거, 토글)
- SFX: 발자국, 문 열기, 상호작용 효과음

## WebGPU AI 통합 (선택적, opt-in)

### 아키텍처
```
게임 진입
  → 사전 작성된 NPC 대화로 즉시 플레이 (0 다운로드)
  → 로비 AI NPC 접근 시:
     → "AI 모드를 활성화하시겠습니까? (570MB 다운로드)"
     → 동의 시: Web Worker에서 Qwen3-0.6B 다운로드 + 캐싱
     → 포트폴리오 정보를 system prompt로 제공
     → 자유 질문에 동적 응답
     → IndexedDB 캐시 → 재방문 시 즉시 사용
```

### 우선순위
| 단계 | 기능 | 다운로드 |
|------|------|---------|
| v1 (필수) | 사전 작성 대화 | 0 |
| v2 (선택) | Qwen3-0.6B AI 챗 | 570MB |
| v3 (나중) | Kokoro TTS 음성 | 86MB |

### 미지원 브라우저
- WebGPU 미지원 시 WASM fallback 자동 전환 (느리지만 동작)
- AI 기능 자체가 opt-in이므로 핵심 경험에 영향 없음

## 레이어 구조

```
[React DOM]       ← 대화창, 상세 패널, 미니맵, 모바일 조작 UI
[PixiJS Canvas]   ← 타일맵, 캐릭터, 오브젝트, 파티클, 조명
[Next.js Page]    ← 데이터 페칭 (projects.json, about.json, MDX)
[Web Worker]      ← AI 추론 (선택적, UI 블로킹 없음)
```

## 네비게이션 통합

- 홈페이지 히어로 섹션에 "Explore my world" CTA 버튼
- 네비게이션 메뉴에 "Play" 항목 (컨트롤러 아이콘)
- 기존 i18n 구조 유지 (`/[locale]/play`)

## 진입 경험

1. 간단한 로딩 화면 — 에셋 로드 중 픽셀아트 로딩바
2. 인트로 안내 — "방향키로 이동, Space로 상호작용"
3. 로비에서 시작 — NPC가 자연스럽게 가이드
