# Next.js로 이중 언어 블로그 만들기

개발자 포트폴리오 블로그를 운영하면서, 한국어와 영어 두 가지 언어로 콘텐츠를 제공하고 싶었습니다. 한국어 사용자와 영어 사용자 모두에게 도달할 수 있는 블로그를 만들기 위해 이중 언어 시스템을 구축했습니다.

## 왜 이중 언어인가?

개발자로서 기술 블로그를 운영할 때, 언어의 장벽은 큰 제약입니다. 한국어로만 작성하면 글로벌 독자에게 도달하기 어렵고, 영어로만 작성하면 한국어 커뮤니티와의 소통이 줄어듭니다.

## 기술 스택

이중 언어 블로그를 구현하기 위해 다음 기술을 사용했습니다:

- **Next.js App Router**: `[locale]` 동적 라우팅으로 언어별 페이지 분리
- **next-intl**: UI 번역 및 라우팅 관리
- **Keystatic CMS**: 언어별 컬렉션으로 콘텐츠 관리
- **디렉토리 기반 분리**: `content/posts/ko/`와 `content/posts/en/`

## SEO 최적화

이중 언어 블로그에서 SEO는 특히 중요합니다:

1. **hreflang 태그**: 각 페이지에 언어별 대체 URL을 명시
2. **사이트맵 alternates**: 검색 엔진이 언어 버전을 올바르게 인덱싱
3. **독립적 메타데이터**: 각 언어 버전마다 고유한 title, description, OG 이미지

## 번역 워크플로우

AI를 활용한 번역 워크플로우를 구축했습니다. Claude Code를 사용하여 미번역 포스트를 확인하고, 한국어 원본을 영어로 번역합니다. API 비용 없이, 기존 구독만으로 고품질 번역이 가능합니다.

## 결론

이중 언어 블로그는 더 넓은 독자층에게 도달할 수 있는 강력한 도구입니다. Next.js의 App Router와 next-intl의 조합은 이를 구현하기에 훌륭한 기술 스택입니다.
